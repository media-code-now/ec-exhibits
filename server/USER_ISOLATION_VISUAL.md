## How It Works

### 1. Database Level

```sql
ALTER TABLE projects 
ADD COLUMN user_id UUID REFERENCES users(id) ON DELETE CASCADE;
                    â–²                         â–²
                    â”‚                         â”‚
            Foreign Key Constraint    Delete projects when user deleted
```

### 2. Application Level - GET Route

```javascript
// User 1 logs in â†’ req.user.id = "user-1"
app.get('/projects', authRequired, async (req, res) => {
  const projects = await prisma.project.findMany({
    where: {
      userId: req.user.id  // â† "user-1"
    }
  });
  // Returns: [Project A, Project C]  â† Only user-1's projects
});

// User 2 logs in â†’ req.user.id = "user-2"
// Returns: [Project B]  â† Only user-2's projects
```

### 3. Application Level - POST Route

```javascript
// User 1 creates a project
app.post('/projects', authRequired, async (req, res) => {
  const project = await prisma.project.create({
    data: {
      name: req.body.name,
      userId: req.user.id  // â† Automatically set to "user-1"
    }
  });
  // Created project automatically belongs to user-1
});
```

## Before vs After Comparison

### Before: Insecure

```javascript
// âŒ Anyone can see all projects

  const projects = projectStore.getAll();
  res.json({ projects });  // Returns ALL projects
});

// âŒ No authentication required
// âŒ No user_id set

  const project = projectStore.create({
    name: req.body.name
    // Missing: userId
  });
  res.json({ project });
});
```

### After: Secure

```javascript
// âœ… Authentication required
// âœ… Only returns user's projects

  const projects = await prisma.project.findMany({
    where: {
      userId: req.user.id  // â† Filter by authenticated user
    }
  });
  res.json({ projects });
});

// âœ… Authentication required
// âœ… user_id automatically set

  const project = await prisma.project.create({
    data: {
      name: req.body.name,
      userId: req.user.id  // â† Automatically assigned
    }
  });
  res.json({ project });
});
```
